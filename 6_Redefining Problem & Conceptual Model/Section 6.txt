Information Architecture
--> satu effort buat nyusun listig menu yg kira kira akan hadir di aplikasi
bedanya sm user flow
- user flow perjalanan dri awal login smpe menyelesaikan maintask utama aplikasi (klo ecommner main tasknya oembelian) 
- IA semua menu-menu yg bisa user akses ketika buka app apa aja 
- IA diapakai untuk displaying breadcrumb menu (menavigasi app yg dipunya) 
- IA isinya berupa navigasi menu, 
- Userflow perjalanannya, IA yg bisa divisit di app itu apa apa aja 

CARA SUSUN IA 
- riset yg dipake courtsorting ; karena kita bikin app ga sendiri kan ada tim riset, ui, business, ada useer j
- sedian sticky notes, kasih ke user klo menu dashboard ada apa aja, profil ada apa aja 
- IA melalui courtsorting bakal keliatan urutan yg paling umum mana 
- IA bisa tanpa pake courtsorting, ngegunain past research. atau fieldresearch yg di lapanagan penyusuan IA nya seperti apa. Cara mengkonfimasi nya kasih ke user bener ga kaya gini, klo kurang susunannya user kasih kesempatan utk malkukan courtsorting. 
- user flow dtgnya dari kebiasaan user, tp biasanya bisa dtg juga dari user journey. klo gaada userjourney ttp dari kebiasaan user
