---> What does it have to do with UX UI?

c. Framework 101 (Fun fact design framework)
---> How it should work? Pentingnya framework
	1. Membuat step harus ngapain utk selesain problem
	2. Jelas atau paham dulu goals dari plan/step yg udah dibikin 
	3. Ngebuat 1 framework (konsep) tapi maksimalkan dgn baik 
	4. Dari 3 itu membantu untuk mencapai kesepakatanutk para ux ui
	5. Oleh krn itu framework juga nge save waktu, jadi ga effort banget 
	6. Minimize misunderstanding tp buat org org yg udh berpengalaman di framework nya (krn klo first time wajar ada misunderstanding) 
	7. FINALLY THEY WORK YEYYYY \^-^/ 
--> Usability Relevancy (dijelasin scr singkat klo lengkapnya google aja)
	- Usability (hasil akhir setelah nge developing (udh ad aproblem solusi, tpi hasil solusi belum bisa nge solve problen yg udah ada) nah itu guna asability)
	- Diukur sebarapa efektifkah solusi kita dapat menyelesaikan problem user, nah pengukurannya itu = usability 
	- Salah satu parameternya : succes rate (succes rate dri solusi yg ditawarkan seberapa gede?? ) 

d. Importance of Framework 
	- Memudahkan mendesain proses secara step by step 
	- Framework membuat kita mendapatkan semacam penghargaan dri setiap produk yg udah dibikin/tawarkan framework bantu utk mengembangkan produk 
	- Kreatifitas dilatih (misal ide, diskusi, dll) 
e. Framework Diversity di UX UI
	1. Understanding 5 elements of UX design 
	- Microsoft, gooogle make ini
	- Dari setiap layar yg ditampilkan (gambar sebelah kanan) beru[a set of decision (surface - strategy) 
	- bacanya dari bawah ke atas (strategy - surface) 
	- Strategy : decision yg dimana kita harus memahami kebutuhan user dan tujuan bisnis 
	- Scope : mendefine apa sih yg ingin kita bikin berdasarkan kebutuhan user (memetakan semua problem user disini)--> fokus ke prblem statement 
	- Structure : proses yg awalnya define menuju ke olusinya --> Fokus ke solution statement
	  Media jalam menuju fase solusi. 
	- Skeleton : teknis solusinya gimana berdasarkan problem yg idh dipetakan (scope), struktur lebih ke gmn caranya solusi designnya gimana, skeleton teknisnya gimana layoutingnya gimana 
	- Bedanya struktur skeleton 
	Struktur : ngeprovide solusi apa aja yg bakal ditaruh designnya gimana outlinenya gimana 
        Skeleton : Teknis, o desainnya coba deh kek gini, layoutingnya taro yg kek gini 
	- Outputnya surface 
	
	2. UCD : User Centered Design 
	- lebih ke sifatnya ke user (placing user as the center of design decision) 
	- Ada 4 fase, setiap fase ada kemiripan sama yg 5 element
	- Fase evaluate ada yg balik ke design solutions dll jdi yaa dinamis juga ga harus runtut tergantung sikon dan kebutuhan 

	3. The infamous Design thinking (SERING DIPAKE) 
	- Sering dipakai karena dri setiap fase semuanya runtut, tapi sifatnya design thinkin ini bukan yg arus mulai dari emphatize sampai  ke implement, jadi ga harus ngurut karena sifatnya DINAMIS 

	4. Double Diamond 
	- Cek gambar yg 5 element, ada persamaan sama yg ini. 

	5. Lean UX (FW BARU)
	- Kadang lebih ringkas drpd framework lain karena fw yg mintanya cepet tapi ga menutup kemungkinan gabakal mengurangi kalitas developing 
	- lebih cepet krn fokus ke setelah produknya jadi next stepnya bakal ngapain lagi, klo proses mirip sama ucd, lebih cocok digunakan utk org yg makai AJAIL/AGILE?? SPRINT?

	6. Hook Model (FW BARU)
	- Lebih ke psychology usernya, ketika user berinteraksi dgn produk gimana perasaan mereka? kenapa ngerasa gitu? apa motiv ereka? ketika berinteraksi sm produk merreka merasa apa sedih seneng frustasi itu dicatet di fw ini 
	- observing user, 
	- fokus ke usability juga 
	- apps yg udh ada di tes ke user dan diliat behaviornya gimana ada trigger ada action, action ada variable reward 	 

---> Perbandingan Design Thinking dan UCD 
UCD : Fokus ke bisnis dan user 
Design Thinking : kebutuhan setiap user kaya gimana
